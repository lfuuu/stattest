Инструкция по установке ПО Welltime


1.    Системные требования
К серверу для установки ПО Welltime предъявляются следующие требования:
•    Процессор, поддерживающий архитектуру Intel i686, тактовая частота не менее 800 МГц;
•    Объём оперативной памяти не менее 512 МБ;
•    Системная логика (материнская плата) — поддерживаемая ОС CentOS 6
•    Жёсткий диск объёмом не менее 10 ГБ (потребление дискового пространства зависит от объёма записей разговоров и времени их хранения);
•    Сетевой интерфейс — 10/100/1000BASE-T Ethernet, поддерживаемый ОС CentOS 6;
•    Интерфейс USB 1.1 или 2.0.
Также, только для установки ПО Welltime необходимы:
•    DVD-привод;
•    Графический терминал (монитор и клавиатура).
Примечание: В случае установки ПО Welltime на виртуальный сервер, необходимо обеспечить проксирование аппаратного USB-ключа HASP на виртуальную машину.
2.    Установка ПО Welltime
Установка ПО Welltime на сервер производится следующим образом:
1.    Вставьте компакт диск с копией ПО Welltime в DVD-привод сервера.
2.    В настройках BIOS или загрузочном меню выберите загрузку с привода компакт-дисков.
3.    После загрузки с диска, на экране появится меню с несколькими вариантами установки:
•    Автоматическая установка Welltime на первый жёсткий диск сервера;
•    Автоматическая установка Welltime на программный RAID-1 массив из первых двух жестких дисков сервера;
•    Установка Welltime в ручном режиме .
4.    При выборе автоматической установки, программа самостоятельно разметит жёсткие диски сервера и установит операционную систему, ПО Welltime и всё необходимое дополнительное ПО. Установка займёт 10-15 минут, в зависимости от конфигурации сервера.
Внимание! Программа установки удалит всё содержимое с диска, на который производится установка системы.
5.    После завершения установки, программа попросит перезагрузить сервер. После перезагрузки, выключите в настройках BIOS загрузку с привода компакт-дисков и загрузитесь с первого жёсткого диска сервера.
6.    После завершения загрузки ОС, войдите в систему с терминала под пользователем root и паролем 'welltimeroot'. После входа автоматически запустится программа первоначальной настройки ПО Welltime,  в которой необходимо задать настройки сетевых интерфейсов сервера, а также ввести новый пароль пользователя root.
7.    После завершения первоначальной настройки, перезагрузите сервер, введя команду 'reboot'. После загрузки операционной системы ПО Welltime готово к настройке с использованием веб-интерфейса.
8.    Документацию по настройке и работе с ПО Welltime вы можете найти на сайте http://wiki.welltime.ru
3.    Настройки безопасности
1.    Смените установленный по умолчанию пароль пользователя admin.
2.    Запретите удаленный доступ по SSH системному пользователю root и используйте стойкие (сложные) пароли для остальных пользователей.
3.    Используйте стойкие пароли для всех учетных записей в системе  ПО Welltime (как для агентов, так и для портов). ПО Welltime может самостоятельно генерировать достаточно стойкие пароли при создании нового агента или порта.
4.     После завершения начальной настройки системы, обязательно измените конфигурацию iptables, желательно, по принципу "белого списка" (запрещено все, кроме того, что разрешено). Особенно следует обратить внимание на адреса и подсети, с которых можно получить доступ к Welltime по SSH, HTTP и SIP (порты tcp/22, tcp/80, udp/5060) - этот список желательно сократить до минимума. Если вы испытываете затруднения в работе с iptables - обратитесь к службе технической поддержки Welltime по телефону (495) 950-56-79.
5.    Помимо настроек межсетевого экрана, используйте правила ACL - с помощью них можно определить разрешенные адреса и подсети, с которых разрешен доступ по протоколу SIP к Welltime. В общих настройках находятся глобальные правила ACL, которые применяются ко всем портам в системе. В настройках портов можно создать правила ACL, применяющиеся только к данному порту. Внимание! Локальные настройки имеют приоритет перед глобальными.
6.    Включение в настройках меню набора добавочного номера в IVR групп правил исходящей маршрутизации может быть серьезной уязвимостью, так как позволяет внешнему абоненту совершить исходящий звонок через вашу АТС. Номерные ёмкости операторов регулярно прозваниваются с целью обнаружения подобных уязвимостей, поэтому лучше отказаться от использования данного функционала. В качестве альтернативы, вы можете воспользоваться функцией "Callback", позволяющей указать список номеров, которым разрешено использование АТС для совершения исходящих вызовов снаружи, или функцией ветвления IVR в зависимости от входящего номера, позволяющей разрешить донабор исходящего номера по принципу "белого списка"
7.    Еще одним средством обеспечения безопасности системы Welltime, помимо списков ACL и межсетевого экрана, является системная служба fail2ban, которая анализирует журналы безопасности сервера и блокирует доступ к серверу с IP-адресов, проявляющих аномальную активность (а именно, множество неудачных попыток залогинивания в систему под SSH или SIP-аккаунтом за короткий промежуток времени). Не отключайте и не удаляйте эту службу.

В случае возникновения каких-либо проблем или вопросов, связанных с установкой ПО Welltime, обращайтесь в службу технической поддержки Welltime по телефону (495) 950-56-79 или по электронной почте support@welltime.ru
