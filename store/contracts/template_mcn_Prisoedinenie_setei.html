2. ОПРЕДЕЛЕНИЯ

2.1. Договор - означает настоящий договор со всеми Приложениями и дополнительными соглашениями к нему.

2.2. ТУ – технические условия на присоединение сетей Сторон.

2.3. Услуга присоединения – деятельность, направленная на удовлетворение потребности операторов связи в организации взаимодействия сетей  передачи данных, при которой становятся возможными установление соединения и передача данных между пользователями взаимодействующих сетей передачи данных. Услуга присоединения включает в себя:

· Монтаж и наладка средств связи, образующих точку присоединения.

· Присоединение сети связи Сторон.

2.4. Точка присоединения – средства связи МСН Телеком/Оператора,  с помощью которых осуществляется подключение средств связи сети электросвязи МСН Телекома/Оператора и обеспечивается возможность пропуска трафика между этими сетями.

2.5. Пропуск трафика означает осуществление процесса установления соединения и передачи нагрузки, создаваемой потоком вызовов, сообщений и сигналов, поступающих от сети передачи данных одной Стороны на сеть передачи данных другой Стороны.

1. 2.6. Услуга по пропуску трафика – деятельность, направленная на удовлетворение потребности операторов связи в Пропуске трафика между взаимодействующими сетями передачи данных, при которой становится возможным оказание Сторонами услуг связи по передаче данных. Перечень оказываемых Услуг по пропуску трафика определяется в соответствии с Приложением № 3.

2.7. Отчетный месяц – период продолжительностью в один календарный месяц, в котором соответствующие услуги были оказаны.

2.8. Расчетный месяц – период продолжительностью в один календарный месяц, следующий за Отчетным периодом.

Положение по оказанию услуги  по пропуску трафика.

1.      Термины и определения.

Трафик – совокупность цифровых данных, передаваемых по Сети.

Направление – географическое расположение точек, на которые передается трафик.

Код – цифровое значение, соответствующее направлению пропуска трафика. Используется для тарификации.

2.      Описание услуги.

В рамках предоставления Услуги по пропуску трафика каждая из Сторон осуществляет двунаправленную передачу данных другой Стороны по своей сети по различным направлениям через организованные точки присоединения сетей Сторон. Взаимодействие сетей должно осуществляться на базе протоколов IP. Точка присоединения должна иметь способ идентификации, основанный на формате ABC@IP_ADDRESS, где:

·         АВС – код географически определяемой зоны нумерации в соответствии с Российской системой и планом нумерации телефонной сети связи общего пользования;

·         IP_ADDRESS - адрес IP сети.

Стороны тарифицируют трафик в зависимости от кода, соответствующего направлению пропуска трафика от точки присоединения.

Для корректной тарификации Оператор обязан маркировать пакеты данных по протоколу IP кодами направлений, при этом:

- передача голосового трафика осуществляется:

·         на базе SIP-протоколов в соответствии с рекомендациями IETF RFC 3261 “SIP: Session Initiation Protocol”, June 2002;

- формат адреса вызываемого абонента должен иметь вид Е.164@IP_ADDRESS, где:

·         Е.164 – формат телефонного номера в соответствии с рекомендацией ITU-T E.164 “The International public telecommunication numbering plan”, 1997;

·         IP_ADDRESS – адрес сети либо из адресного плана публичной сети Интернет, соответствующим образом оформленного в RIPE NCC и принадлежащего оператору, на которого направляется трафик, либо из частного адресного плана.

3.      Параметры услуги:

·         поддерживаемые на сети кодеки в соответствии с рекомендациями ITU-T:

o   G.711 “lossless compression scheme”, 1972;

·         передача сигналов DTMF – в соответствии с рекомендациями IETF RFC 2833 “RTP Payload for DTMF Digits, Telephony Tones and Telephony Signals”, 2000;

·         поддержка факсов – в соответствии с рекомендациями ITU-T T.38 “Procedures for real-time Group 3 facsimile communication over IP networks”, 1998, 2007.

·   нормативными актами (правилами, руководствами, инструкциями, приказами, распоряжениями и проч.) государственных органов, изданными ими в пределах их компетенции и утвержденными в порядке, установленном в Российской Федерации.

2.    Официальное уведомление от одной из Сторон о необходимости проведения сверки вместе с данными для сверки предоставляются другой Стороне не позднее 5 (пяти) календарных дней после получения данных согласно разделу 6 настоящего договора. Непредставление в указанный срок официального уведомления и данных для сверки  будет считаться отказом Стороны от проведения сверки.

3.    При проведении сверки Стороны предоставляют друг другу данные, используемые при расчетах. Предоставленные для сверки данные являются неполными, если в них отсутствует часть вошедших в окончательный расчет учетных записей, относящихся к согласованному Сторонами периоду времени.

4.    Сторона, обнаружившая, что предоставленные для сверки данные являются неполными, направляет другой Стороне запрос о предоставлении полных данных не позднее трех рабочих дней после получения неполных данных. Сторона, предоставившая неполные данные, в течение двух рабочих дней после получения запроса предоставляет полные данные для сверки. В случае отказа Стороны, предоставившей неполные данные, предоставить для сверки полные данные в течение двух рабочих дней после получения запроса от Стороны, проводящей сверку, сверка считается невыполненной по вине Стороны, предоставившей неполные данные.

5.    Сторона, по расчетам которой получилась меньшая сумма к оплате по данному направлению, является ответственной за проведение сверки. Сторона, ответственная за проведение сверки, должна предоставить результаты сверки другой Стороне не позднее пяти рабочих дней после получения полных учетных данных.

6.    Если в результате сверки Сторонам не удается уменьшить величину расхождения данных по трафику до величины в 3% и менее, а также, если одна из Сторон не признает результатов сверки или  одна  из Сторон отказалась предоставить данные по трафику для проведения сверки, Стороны действуют в соответствии с разделом 10 настоящего Договора.

7.    В случае если одна из Сторон не имеет возможности предоставить данные для сверки, то по взаимному согласию Сторон для расчетов используются данные другой Стороны.

8.    По результатам проведения сверки данных по пропущенному трафику Стороны подписывают Акт о результатах сверки, который является основанием для проведения перерасчета.
