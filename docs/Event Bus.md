В стате реализована работа с очередью сообщений из кафки для асинхронного выполнения команд. 
В данный момент, в первой версии - это обертка над API-методами стат.

Топик: `event-bus-cmd`

В теле сообщения передается json со след полями:
`version` - версия протокола взаимодействия. Сейчас поддерживается - 1.
`type` - тип сообщения. (см. типы сообщений)
`id` - идентификатор запроса/ответа. Так же это значение является ключом сообщения.
`dst` - кому адресована команда
`src` - кто отправляет. В ответе значения `dst` и `src` меняются местами.

Типы сообщений:
* `cmd` - команда на выполнение. Обязательные поля в сообщении:
  - `cmd` - точка вызова API (`/api/internal/uu/get-tariffs` etc.)
  - `argv` - hashmap параметров вызова функции.

* `cmd_result` - результат выполнения команды.
  - `result` - результат вызова API-метода.
